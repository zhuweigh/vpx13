b¶

Set socket buffer size. If used, should be used with roughly +100 then the datalen (-s option). The default value is 8192.<br/>Minimum length = 132<br/>Maximum length = 131071.

c¶

Number of packets to send. The default value is infinite. For Nitro API, defalut value is taken as 5.<br/>Minimum length = 1<br/>Maximum length = 65535.

i¶

Waiting time, in seconds. The default value is 1 second.<br/>Maximum length = 65535.

I¶

Network interface on which to ping, if you have multiple interfaces.<br/>Minimum length = 1<br/>Maximum length = 15.

m¶

By default, ping6 asks the kernel to fragment packets to fit into the minimum IPv6 MTU.The -m option will suppress the behavior for unicast packets.

n¶

Numeric output only. No name resolution.

p¶

Pattern to fill in packets. Can be up to 16 bytes, useful for diagnosing data-dependent problems.<br/>Minimum length = 1<br/>Maximum length = 32.

q¶

Quiet output. Only summary is printed. For Nitro API, this flag is set by default.

s¶

Data size, in bytes. The default value is 32.<br/>Maximum length = 65527.

V¶

VLAN ID for link local address.<br/>Minimum length = 1<br/>Maximum length = 4094.

S¶

Source IP address to be used in the outgoing query packets.<br/>Minimum length = 1.

T¶

Traffic Domain Id.<br/>Minimum length = 1<br/>Maximum length = 4094.

t¶

Timeout in seconds before ping6 exits.

hostName¶

Address of host to ping.<br/>Minimum length = 1<br/>Maximum length = 256.
