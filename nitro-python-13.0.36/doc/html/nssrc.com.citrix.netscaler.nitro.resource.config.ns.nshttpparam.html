nshttpparam module¶

class nshttpparam¶

Bases: nssrc.com.citrix.netscaler.nitro.resource.base.base_resource.base_resource

Configuration for HTTP parameter resource.

dropinvalreqs¶

Drop invalid HTTP requests or responses.<br/>Default value: OFF<br/>Possible values = ON, OFF.

markhttp09inval¶

Mark HTTP/0.9 requests as invalid.<br/>Default value: OFF<br/>Possible values = ON, OFF.

markconnreqinval¶

Mark CONNECT requests as invalid.<br/>Default value: OFF<br/>Possible values = ON, OFF.

insnssrvrhdr¶

Enable or disable Citrix ADC server header insertion for Citrix ADC generated HTTP responses.<br/>Default value: OFF<br/>Possible values = ON, OFF.

nssrvrhdr¶

The server header value to be inserted. If no explicit header is specified then NSBUILD.RELEASE is used as default server header.<br/>Minimum length = 1.

logerrresp¶

Server header value to be inserted.<br/>Default value: ON<br/>Possible values = ON, OFF.

conmultiplex¶

Reuse server connections for requests from more than one client connections.<br/>Default value: ENABLED<br/>Possible values = ENABLED, DISABLED.

maxreusepool¶

Maximum limit on the number of connections, from the Citrix ADC to a particular server that are kept in the reuse pool. This setting is helpful for optimal memory utilization and for reducing the idle connections to the server just after the peak time.<br/>Default value: 0<br/>Maximum length = 360000.

http2serverside¶

Enable/Disable HTTP/2 on server side.<br/>Default value: OFF<br/>Possible values = ON, OFF.

feature¶

The feature to be checked while applying this config.

ignoreconnectcodingscheme¶

Ignore Coding scheme in CONNECT request.<br/>Default value: DISABLED<br/>Possible values = ENABLED, DISABLED.
