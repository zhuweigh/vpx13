name¶

Name for the new login schema. Login schema defines the way login form is rendered. It provides a way to customize the fields that are shown to the user. Must begin with an ASCII alphanumeric or underscore (_) character, and must contain only ASCII alphanumeric, underscore, hash (#), period (.), space, colon (:), at (@), equals (=), and hyphen (-) characters. Cannot be changed after an action is created. The following requirement applies only to the Citrix ADC CLI: If the name includes one or more spaces, enclose the name in double or single quotation marks (for example, “my action” or ‘my action’).<br/>Minimum length = 1.

authenticationschema¶

Name of the file for reading authentication schema to be sent for Login Page UI. This file should contain xml definition of elements as per Citrix Forms Authentication Protocol to be able to render login form. If administrator does not want to prompt users for additional credentials but continue with previously obtained credentials, then “noschema” can be given as argument. Please note that this applies only to loginSchemas that are used with user-defined factors, and not the vserver factor.<br/>Minimum length = 1.

userexpression¶

Expression for username extraction during login. This can be any relevant advanced policy expression.<br/>Minimum length = 1.

passwdexpression¶

Expression for password extraction during login. This can be any relevant advanced policy expression.<br/>Minimum length = 1.

usercredentialindex¶

The index at which user entered username should be stored in session.<br/>Minimum length = 1<br/>Maximum length = 16.

passwordcredentialindex¶

The index at which user entered password should be stored in session.<br/>Minimum length = 1<br/>Maximum length = 16.

authenticationstrength¶

Weight of the current authentication.<br/>Maximum length = 65535.

ssocredentials¶

This option indicates whether current factor credentials are the default SSO (SingleSignOn) credentials.<br/>Default value: NO<br/>Possible values = YES, NO.

feature¶

The feature to be checked while applying this config.
