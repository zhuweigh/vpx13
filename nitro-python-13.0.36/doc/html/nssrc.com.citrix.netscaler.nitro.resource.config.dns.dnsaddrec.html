class dnsaddrec¶

Bases: nssrc.com.citrix.netscaler.nitro.resource.base.base_resource.base_resource

Configuration for address type record resource.

hostname¶

Domain name.<br/>Minimum length = 1.

ipaddress¶

One or more IPv4 addresses to assign to the domain name.<br/>Minimum length = 1.

ttl¶

Time to Live (TTL), in seconds, for the record. TTL is the time for which the record must be cached by DNS proxies. The specified TTL is applied to all the resource records that are of the same record type and belong to the specified domain name. For example, if you add an address record, with a TTL of 36000, to the domain name example.com, the TTLs of all the address records of example.com are changed to 36000. If the TTL is not specified, the Citrix ADC uses either the DNS zone’s minimum TTL or, if the SOA record is not available on the appliance, the default value of 3600.<br/>Default value: 3600<br/>Maximum length = 2147483647.

feature¶

The feature to be checked while applying this config.

ecssubnet¶

Subnet for which the cached address records need to be removed.

type¶

The address record type. The type can take 3 values: ADNS - If this is specified, all of the authoritative address records will be displayed. PROXY - If this is specified, all of the proxy address records will be displayed. ALL - If this is specified, all of the address records will be displayed.<br/>Possible values = ALL, ADNS, PROXY.

nodeid¶

Unique number that identifies the cluster node.<br/>Maximum length = 31.

vservername¶

Vitual server name.

authtype¶

Authentication type.<br/>Possible values = ALL, ADNS, PROXY.
